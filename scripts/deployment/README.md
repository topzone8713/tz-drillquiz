# Deployment Scripts

애플리케이션 배포 및 운영을 위한 스크립트들입니다.

## 📁 Scripts

### `start.sh`
- **목적**: 개발 환경 서버 시작
- **사용법**: `./start.sh`
- **기능**: 
  - Django 백엔드 서버 시작 (포트 8000)
  - Node.js 프론트엔드 서버 시작 (포트 8080)
  - 데이터베이스 연결 확인

### `stop.sh`
- **목적**: 개발 환경 서버 중지
- **사용법**: `./stop.sh`
- **기능**: 
  - 실행 중인 서버 프로세스 종료
  - 포트 해제

### `k8s.sh`
- **목적**: Kubernetes 배포 관리
- **사용법**: `./k8s.sh [dev|qa|prod]`
- **기능**: 
  - 환경별 Kubernetes 매니페스트 적용
  - 배포 상태 확인
  - 롤백 지원

## 🚀 배포 환경

- **Development**: 로컬 개발 환경
- **QA**: 테스트 환경
- **Production**: 운영 환경

## ⚠️ 주의사항

- **권한**: 프로덕션 배포는 승인된 사용자만
- **테스트**: QA 환경에서 먼저 테스트
- **백업**: 배포 전 데이터 백업

## 🔄 배포 순서

1. 코드 검증
2. 테스트 실행
3. 백업 생성
4. 배포 실행
5. 상태 확인
6. 롤백 준비
